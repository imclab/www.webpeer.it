Webpeer To Help Ubuntu Server

24 Aug 2010

Did you ever upgrade your computer on a release day ? I bet you did, we all do. This is why it is so slow. The servers bandwidth is limited and we are all hogging it. It could not be a more frustrating moment for slowness, i’m all excited about the new release and it is coming slower than a lazy snail. Wouldn’t it be nice to share the packages i already got with other ubuntu members and get last ubuntu sooner thanks to faster download ?

Well i got a good news. WebPeer makes it possible in less than 5min. For that, we gonna use webpeer-http-proxy. It is an http proxy using binding for nodejs we described in a previous post. Once you installed it, run the following. This will simply reroute .deb files thru webpeer, and leave the rest alone.

node webpeer-http-proxy.js .deb

Now that the http proxy is running, lets try an apt-get upgrade

sudo http_proxy=http://127.0.0.1:8080 apt-get upgrade

That’s all. nice no ? If you like it, you can make it permanent by creating an apt configuration file. lets call it /etc/apt/apt.conf.d/80webpeer and put the following content to it.

Acquire::http::Proxy "http://localhost:8080";

If you want to restart webpeer-http-proxy after a reboot, put it in a service.
